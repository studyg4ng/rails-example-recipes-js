Cooking on Rails Exercise

Learn about the asset pipeline

About your application’s environment

What to do

  1. Load seed data

    What is seed data? Where is the actual file located? And what was that rake task to load it?

  2. Create HTML

    Study the draft: which parts of the html go into the layout? the view? a partial? Study the layout that's already there: which parts do you need to preserve?

    This step should be a git commit!

  3. Move CSS and img over to Asset Pipeline

    Which part of the existing Stylesheet should be turned into sass? Use convert-sass on the command line to do it. Which part should remain plain css? Just copy it over. Don't forget to test in your browser!

    This step should be a git commit!

  4. Assets in production

    Now check out your app a second time from git, and run it in production. A few things are different:

    • Rake tasks: add RAILS_ENV="production" at the end of the command.
    • Rails server: rails s -e production.
    • Rails console: rails c production.

    If you use only webrick: set config.serve_static_assets = true. You could try apache + mod_passenger instead.

    Where are the compiled assets now?

  5. Javascript, jQuery, AJAX

    Back in development: Watch Railscast #136 jQuery & Ajax (revised). Use what you learn there to make adding an ingredient while creating a new Recipe easier. (load and send the new_ingredient form with ajax)
